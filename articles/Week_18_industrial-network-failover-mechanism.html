Industrial Network Failover Mechanism

5 min read
🇮🇩 ID🇬🇧 EN

Detik ke-23. PLC kehilangan koneksi ke HMI. Detik ke-27. Conveyor berhenti. Detik ke-45. Supervisor menelepon teknisi.

Satu link fiber putus — tertabrak forklift di area gudang. Seharusnya ini bukan masalah. Seharusnya jaringan punya jalur cadangan. Tapi switch tidak dikonfigurasi untuk failover. Tidak ada ring. Tidak ada redundant link. Tidak ada alternate path.

Industrial network failover adalah perbedaan antara insiden kecil dan shutdown total. Artikel ini membahas mekanisme failover di jaringan industri — dari protokol layer 2 hingga strategi arsitektur.

Mengapa Failover di Jaringan Industri Berbeda

Failover di jaringan kantor bisa memakan waktu 30 detik. TCP akan retransmit. Email tertunda. Tidak ada yang mati.

Failover di jaringan industri harus terjadi dalam milidetik. IACS (Industrial Automation and Control System) memiliki batas toleransi interrupt kurang dari 100 ms (Hanes et al., 2017). Di atas itu, PLC menganggap link putus. SCADA kehilangan data real-time. Proses produksi terganggu.

Bukan hanya soal kecepatan — tapi juga determinisme. Failover di jaringan industri harus predictable. Engineer harus tahu persis jalur mana yang akan diambil saat link primer gagal. Tidak boleh ada kejutan.

Protokol Failover: Dari STP ke ERPS

Spanning Tree: Fondasi yang Lambat

Spanning Tree Protocol (STP, IEEE 802.1d) adalah protokol failover tertua di Layer 2. Mekanismenya sederhana: deteksi loop → blokir port redundant → saat link aktif gagal → buka port yang diblokir.

Masalahnya: STP butuh 30-50 detik untuk konvergensi. Tiga puluh detik adalah selamanya di lantai produksi.

Rapid Spanning Tree Protocol (RSTP, IEEE 802.1w) memperbaiki ini. Konvergensi turun ke 1-2 detik. Cukup untuk jaringan enterprise. Masih terlalu lambat untuk otomasi industri.

Ring Protocol: ERPS dan REP

Ethernet Ring Protection Switching (ERPS, ITU-T G.8032) dan Resilient Ethernet Protocol (REP) dirancang khusus untuk topologi ring — topologi paling umum di jaringan industri.

Mekanismenya: semua switch dalam ring terkoneksi membentuk lingkaran. Satu link ditetapkan sebagai Ring Protection Link (RPL) — dalam kondisi normal, link ini diblokir untuk mencegah loop. Saat ada link lain di ring yang gagal, RPL langsung dibuka. Waktu recovery: di bawah 50 ms.

REP bahkan bisa mencapai sub-50 ms pada infrastruktur fiber. Detection mechanism menggunakan loss of signal (LOS) — switch langsung tahu saat fiber putus tanpa perlu menunggu timer hello protocol (Hanes et al., 2017).

LACP Link Aggregation

Link Aggregation Control Protocol (LACP, IEEE 802.3ad) mengambil pendekatan berbeda. Alih-alih menyediakan link backup pasif, LACP menggabungkan beberapa link fisik menjadi satu logical link. Semua link aktif secara simultan. Traffic di-load balance.

Jika satu link gagal, traffic otomatis dialihkan ke link yang tersisa — tanpa konvergensi, tanpa negosiasi ulang. Bandwidth aggregate berkurang, tapi koneksi tidak putus.

LACP ideal untuk uplink antar switch — di mana bandwidth tambahan sama berharganya dengan redundansi.

Dual Homing dan Redundant Star

Tidak semua jaringan industri berbentuk ring. Topologi star dengan dual homing — setiap access switch punya dua uplink ke dua aggregation switch berbeda — juga populer di arsitektur CPwE (Converged Plantwide Ethernet).

Dalam desain ini, RSTP memblokir salah satu uplink. Jika uplink aktif gagal, RSTP membuka uplink cadangan. Kelemahannya: RSTP tetap butuh 1-2 detik. Itulah mengapa beberapa implementasi menggunakan Flex Links — fitur yang langsung mengaktifkan link backup tanpa menunggu konvergensi spanning tree.

Arsitektur vs Protokol

Pemilihan mekanisme failover bukan sekadar memilih protokol tercepat. Setiap desain redundancy failover network punya trade-off yang harus disesuaikan dengan topology dan tolerance downtime pabrik.

Ring + ERPS: recovery <50 ms, tapi semua switch harus support protokol yang sama. Kompleksitas konfigurasi lebih tinggi.

Star + RSTP + LACP: lebih sederhana, bandwidth lebih besar (karena link aggregation), tapi recovery time lebih lambat — 1-2 detik.

Hardware bypass: bukan protokol, tapi mekanisme fisik. Saat switch mati total — power failure, hardware crash — relay bypass menutup sirkuit dan menjaga link tetap physically connected. Traffic dilewatkan tanpa diproses. Ini adalah fail-safe terakhir.

Implementasi di Switch Industrial Modern

Industrial network failover bukan lagi fitur premium. Switch managed industrial modern membawa protokol failover sebagai standar.

ERPS (G.8032) untuk ring topology dengan recovery sub-50 ms. Didukung oleh switch managed seperti seri XPTN-9000 dari Scodeno — bersama dengan RSTP, STP, dan LACP dalam satu perangkat.

RSTP (802.1w) untuk topologi mesh dan star. Konvergensi 1-2 detik — tidak secepat ERPS, tapi lebih fleksibel dalam topologi non-ring.

LACP (802.3ad) untuk link aggregation. Menggabungkan 2-8 link fisik menjadi satu logical trunk. Failover instant — tanpa konvergensi.

Hardware bypass — fitur yang tersedia di switch Bypass Series Scodeno — menjaga kontinuitas physical link bahkan saat switch mengalami complete failure. Traffic tetap mengalir meskipun perangkat mati. Recovery time: nol — karena link tidak pernah benar-benar putus.

Untuk pembahasan lebih mendalam tentang strategi redundancy failover network di arsitektur industri, baca panduan kami tentang failover jaringan industri.

Kesimpulan

Failover bukan fitur tambahan. Ia adalah requirement.

Pilih protokol berdasarkan topology dan tolerance downtime. Ring topology dengan ERPS untuk recovery <50 ms. Star dengan RSTP untuk fleksibilitas. LACP untuk bandwidth + redundansi simultan. Hardware bypass sebagai lapisan terakhir.

Switch industrial modern membawa semua mekanisme ini dalam satu perangkat. Konfigurasikan dengan benar — dan putusnya satu kabel fiber tidak lagi berarti produksi berhenti.
